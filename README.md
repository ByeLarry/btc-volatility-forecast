# Прогнозирование дневной реализованной волатильности BTC

Данный репозиторий содержит реализацию исследования по прогнозированию дневной реализованной волатильности BTC на один день вперед. В работе используются 5-минутные рыночные данные, суточные сетевые показатели блокчейна, оконное формирование обучающих примеров и сравнение моделей MLP и LSTM при одинаковой схеме временной проверки.

## Общая логика

Сначала загружаются исходные 5-минутные ценовые данные BTC (OHLCV) и, при необходимости, суточные сетевые показатели блокчейна. Рыночные данные приводятся к UTC-времени, проверяются на наличие некорректных значений и дубликатов, после чего по 5-минутным логарифмическим доходностям рассчитываются дневная реализованная дисперсия и дневная реализованная волатильность. Неполные UTC-сутки исключаются из набора данных.

Далее рыночные и сетевые данные согласуются по UTC-дате. На основе полученного суточного набора формируются два варианта признаков: базовый, включающий только прошлые значения дневной реализованной волатильности, и расширенный, дополненный метриками блокчейна. Целевая переменная и признаки преобразуются в логарифмическую шкалу, после чего признаки масштабируются отдельно на каждом обучающем интервале, чтобы исключить утечку информации из будущих периодов.

Проверка качества выполняется по схеме скользящего окна. На каждом шаге выделяется обучающий интервал, следующий за ним тестовый интервал, формируются перекрывающиеся временные последовательности заданной длины, затем обучаются модели MLP и LSTM. После получения прогноза выполняется обратное преобразование из логарифмической шкалы, рассчитываются метрики MAE, RMSE и MAPE, а результаты сохраняются в виде таблиц прогнозов, сводных метрик и графиков.

## Основные этапы пайплайна

1. Сбор и загрузка исходных данных.
2. Подготовка 5-минутных рыночных данных и приведение времени к UTC.
3. Расчет дневной реализованной волатильности.
4. Согласование рыночных и сетевых данных по UTC-суткам.
5. Формирование базового и расширенного наборов признаков.
6. Логарифмическое преобразование и масштабирование признаков.
7. Формирование обучающих примеров на основе скользящих окон.
8. Обучение моделей MLP и LSTM.
9. Прогнозирование дневной реализованной волатильности на один день вперед.
10. Расчет метрик качества и сохранение результатов эксперимента.

## Входные данные

Для запуска эксперимента используются CSV-файлы, пути к которым задаются в `config.json`:

- 5-минутные рыночные данные BTC с колонками `timestamp`, `open`, `high`, `low`, `close`, `volume`;
- суточные сетевые показатели с колонками `date`, `unique_addresses`, `transfer_volume_btc`, `avg_fee_usd`.

## Результаты

После выполнения эксперимента сохраняются:

- прогнозы моделей по тестовым интервалам;
- метрики качества по каждому шагу скользящей проверки;
- сводная таблица средних значений метрик;
- графики фактической и прогнозной реализованной волатильности;
- график сравнения качества моделей и наборов признаков.

## Запуск

Перед запуском необходимо подготовить `config.json`, указав пути к данным, параметры окон, список моделей, наборы признаков и параметры обучения.

```bash
python main.py
```

## Описание параметров `config.json`

Все основные параметры эксперимента задаются в файле `config.json`. Конфигурация определяет пути к данным, параметры скользящей временной проверки, состав признаков, список моделей, настройки обучения и правила сохранения результатов.

### `paths`

Раздел задает пути к исходным данным и результатам эксперимента.

| Параметр | Описание |
|---|---|
| `market_5m_csv` | Путь к CSV-файлу с исходными 5-минутными рыночными данными BTC. Файл должен содержать колонки `timestamp`, `open`, `high`, `low`, `close`, `volume`. |
| `network_daily_csv` | Путь к CSV-файлу с суточными сетевыми показателями блокчейна. Используется для расширенного набора признаков `extended`. |
| `daily_dataset_csv` | Путь, по которому сохраняется сформированный итоговый суточный набор данных. |
| `results_dir` | Каталог, в который сохраняются прогнозы, метрики, графики и, при необходимости, обученные модели. |

### `experiment`

Раздел задает основную логику эксперимента и схему временной проверки.

| Параметр | Описание |
|---|---|
| `input_window` | Длина входного окна в сутках. При значении `30` модель использует предыдущие 30 дневных наблюдений для построения прогноза. |
| `forecast_horizon` | Горизонт прогноза в сутках. В текущей реализации используется прогноз на один день вперед, поэтому значение равно `1`. |
| `train_window` | Размер обучающего интервала в сутках на каждом шаге скользящей проверки. |
| `test_window` | Размер тестового интервала в сутках на каждом шаге проверки. При значении `30` модель проверяется на следующих 30 днях после обучающего окна. |
| `step_size` | Шаг сдвига скользящего окна. При значении `30` следующее обучающее и тестовое окно сдвигаются на 30 дней. |
| `feature_sets` | Список наборов признаков для сравнения. `base` использует только прошлые значения реализованной волатильности, `extended` дополнительно использует сетевые показатели блокчейна. |
| `models` | Список моделей, участвующих в эксперименте. В текущей версии используются `mlp` и `lstm`. |
| `scaler` | Способ масштабирования признаков. Значение `standard` соответствует стандартизации, значение `minmax` — масштабированию в заданный диапазон. Масштабирование на каждом шаге настраивается только по обучающему интервалу. |
| `epsilon` | Малая положительная добавка, используемая при логарифмировании и обратном преобразовании, чтобы избежать ошибок при нулевых или слишком малых значениях волатильности. |
| `random_seed` | Фиксированное значение генератора случайных чисел для повышения воспроизводимости эксперимента. |

### `training`

Раздел задает параметры обучения нейросетевых моделей.

| Параметр | Описание |
|---|---|
| `learning_rate` | Скорость обучения оптимизатора Adam. Чем меньше значение, тем осторожнее обновляются веса модели. |
| `batch_size` | Размер мини-пакета наблюдений при обучении. |
| `epochs` | Максимальное количество эпох обучения. Если включена ранняя остановка, обучение может завершиться раньше. |
| `validation_split` | Доля обучающей выборки, используемая для контроля качества во время обучения. |
| `early_stopping` | Включает или отключает раннюю остановку обучения. Если значение `true`, обучение прекращается при отсутствии улучшения ошибки. |
| `early_stopping_patience` | Количество эпох без улучшения, после которого обучение останавливается. |
| `verbose` | Уровень вывода информации во время обучения. Значение `0` отключает подробный вывод. |

### `mlp`

Раздел задает архитектуру модели MLP. Для этой модели входное временное окно предварительно преобразуется в плоский вектор признаков.

| Параметр | Описание |
|---|---|
| `hidden_units` | Количество нейронов в скрытых полносвязных слоях. Например, `[32, 16]` означает два скрытых слоя: первый на 32 нейрона, второй на 16 нейронов. |
| `dropout` | Доля случайно отключаемых нейронов после скрытых слоев. Используется для снижения переобучения. |
| `activation` | Функция активации скрытых слоев. |

### `lstm`

Раздел задает архитектуру модели LSTM, которая обрабатывает входные данные как последовательность дневных наблюдений.

| Параметр | Описание |
|---|---|
| `units` | Количество LSTM-слоев и число нейронов в каждом из них. Например, `[32]` означает один LSTM-слой на 32 нейрона. |
| `dropout` | Доля отключаемых входных связей внутри LSTM-слоя. Используется для регуляризации модели. |
| `recurrent_dropout` | Доля отключаемых рекуррентных связей внутри LSTM-слоя. |
| `dense_units` | Количество нейронов в полносвязных слоях после LSTM-блока. Например, `[8]` означает один дополнительный полносвязный слой на 8 нейронов. |
| `activation` | Функция активации полносвязных слоев после LSTM-блока. |

### `saving`

Раздел определяет, какие результаты сохраняются после выполнения эксперимента.

| Параметр | Описание |
|---|---|
| `save_models` | Если `true`, сохраняются обученные модели для каждого шага проверки, модели и набора признаков. |
| `save_predictions` | Если `true`, сохраняется таблица прогнозов. |
| `save_metrics` | Если `true`, сохраняются метрики качества по каждому шагу проверки и сводная таблица средних метрик. |
| `save_plots` | Если `true`, сохраняются графики фактической и прогнозной реализованной волатильности, а также графики сравнения качества моделей. |
